## Compilação  
A compilação transforma todo o código-fonte em código-objeto antes da execução.  
  
### Etapas  
1. Análise léxica  
2. Análise sintática  
3. Análise semântica  
4. Geração de código  
### Vantagens  
- Alto desempenho  
- Erros detectados antes da execução  
- Código otimizado    
### Desvantagens  
- Menor portabilidade  
- Precisa recompilar  
- Dependência da arquitetura  
## Interpretação  
Na interpretação, o código-fonte é executado diretamente por um interpretador, linha por linha.  
### Vantagens  
- Alta portabilidade  
- Desenvolvimento mais rápido  
- Facilidade de testes   
### Desvantagens  
- Execução mais lenta  
- Erros em tempo de execução  
- Dependência do interpretador    
## Abordagem híbrida  
Nesse modelo, o código-fonte passa por uma pré-compilação, gerando bytecode.  
### Exemplos  
- Java → JVM  
- C# → CLR    
### Vantagens  
- Alta portabilidade  
- Melhor desempenho que linguagens puramente interpretadas  
- Segurança    
### Desvantagens  
- Mais lento que código totalmente compilado  
- Dependência da máquina virtual  
- Maior consumo de memória
